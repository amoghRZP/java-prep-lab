# Java Prep Lab

A structured workspace for improving Java problem-solving, core language concepts, object-oriented design, and interview readiness.

This repository is meant to be a practical learning lab: write code, solve problems, revisit concepts, and keep notes in one place. It is designed for iterative preparation rather than one-time reading.

## Overview

Java Prep Lab is a personal preparation repository focused on building strong fundamentals and consistent practice. It can be used for interview preparation, revision, and hands-on experimentation with Java concepts.

The goal is to keep preparation organized across coding exercises, concept notes, and implementation practice so that progress is easy to track and revisit.

## What This Repository Helps With

- Strengthening core Java fundamentals.
- Practicing problem-solving with clean, testable code.
- Revising object-oriented programming concepts.
- Preparing for coding rounds and technical interviews.
- Building the habit of writing explanations alongside solutions.

## Topics Covered

Depending on how the repository evolves, it can include work across areas such as:

- Java basics: syntax, control flow, methods, arrays, strings.
- Object-oriented programming: classes, inheritance, polymorphism, abstraction, encapsulation.
- Collections framework: List, Set, Map, Queue, Stack, iterators.
- Exception handling and debugging.
- Java 8+ features: streams, lambdas, functional interfaces, optional.
- Concurrency basics: threads, executors, synchronization.
- Problem-solving patterns: two pointers, sliding window, recursion, hashing, sorting, searching.
- Low-level design practice and implementation exercises.
- Notes, revision material, and interview takeaways.

## Repository Philosophy

This repository is built around one idea: preparation improves when concepts and practice live together.

Instead of separating notes, examples, and solutions across different places, this lab keeps them close to the code. That makes it easier to revise a topic, compare approaches, and understand not just what works, but why it works.

## How to Use

1. Pick one topic or problem category.
2. Read or revise the concept briefly.
3. Solve problems or implement examples in Java.
4. Add notes about trade-offs, edge cases, and mistakes.
5. Revisit weak areas until the approach feels natural.

A good workflow is to solve first, then refactor for readability, then add a short explanation of the final approach.

## Running the Code

Use whichever Java workflow matches the repository setup on your machine.

### Option 1: Compile and run with `javac`

```bash
javac ClassName.java
java ClassName
```

### Option 2: Run through an IDE

Open the project in IntelliJ IDEA, VS Code, or Eclipse, then run the required class directly.

### Option 3: Use Maven or Gradle

If the repository later includes a build file such as `pom.xml` or `build.gradle`, use the standard project commands for that setup.

## Recommended Study Flow

- Start with Java fundamentals.
- Move to OOP and collections.
- Practice common coding patterns.
- Add timed problem-solving sessions.
- Revisit weak topics every week.
- Maintain short notes for revision before interviews.

Consistency matters more than volume. Even a small daily streak is more effective than occasional heavy sessions.

## Writing Good Solutions

A strong solution in this repository should aim for:

- Clear naming.
- Correct handling of edge cases.
- Reasonable time and space complexity.
- Short explanation of the approach.
- Optional alternative solution if there is a better trade-off.

For interview preparation, readable code is usually more valuable than clever code.

## Suggested Contribution Style

When adding new problems or examples, try to keep the format consistent:

- Problem statement or title.
- Approach summary.
- Java solution.
- Complexity analysis.
- Edge cases or lessons learned.

That structure makes revision much faster later.
